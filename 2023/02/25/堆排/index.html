经典的排序算法

堆排

堆排:三个 核心的点:
① : heapInsert (从元素 插入点的位置, 向上, 进行 heapInsert )

- 从下向上 构建堆
    - 数组中,一个一个元素的 insert 构建堆的 代价 O(N * lgN)

②: heapify
- 从上向下 (从元素 所在位置, 向下, 进行 heapify)
- 给出一个 数组, 可以倒序构建, 以 时间复杂度 O(N) 的代价 建成 堆 结构!
③: heapSize (逻辑堆的大小)
- 这个 参数也非常的 核心!

时间复杂度 O(N * lgN) 空间复杂度 O(1)

稳定性: 无

堆排： 不稳定： 是因为 会将 堆 头 的元素 和 堆尾的元素 进行交换， 破坏稳定性

快排

时间复杂度 O(n * lg n)

空间复杂度 (lg n) : 递归调用栈的高度!

不稳定

快排不稳定： 是因为 在partaion 的过程中， more 的下一个区域，也就是 more 的左边位置 会与 当前index 位置进行交换， 这一交换就会破坏 稳定性（脑海中过一下这个过程,就明白了!）

归并排序

时间复杂度 O(n * lgn)

空间复杂度 O(n)

稳定: 归并是稳定, 只需要再 merge 的时候 处理相等的时候, 让左边的元素归入数组, 就可以保证 稳定性了!
